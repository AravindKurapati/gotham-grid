# GOTHAM GRID

## Project overview
A retro CRT-aesthetic dashboard that aggregates vibe-coded projects from major cities worldwide. The Bloomberg Terminal of vibe-coding culture. Built with Next.js 14 + React, powered by a hybrid architecture: static pre-fetched data loads instantly, with optional live scanning via Anthropic API + web search.

No Twitter API needed — web search catches indexed tweets, GitHub repos, Reddit posts, HN threads, dev blogs, and deployed demos.

**Name**: GOTHAM GRID
**Tagline**: "Scanning the grid. Tracking the vibe."

## Architecture: Hybrid (Static + Live Scan)

### How it works
1. **Static layer (default)**: At build time (or via `npm run scan`), a Node script calls Anthropic API with web search, fetches projects for each city, saves results as JSON files in `/data/{city}.json`. The site serves this data with zero runtime API calls. Cost: $0 after build.
2. **Live scan layer (optional)**: A "LIVE SCAN" button on the dashboard triggers real-time API search. Protected by:
   - Rate limiting: 5 requests/min per IP via in-memory rate limiter
   - Optional invite code: Set `SCAN_CODE` env var — users must enter it to unlock live scan
   - Anthropic spending cap: Set in console as a safety net
3. **No API key?** The site works perfectly with just the static data. Live scan is a progressive enhancement.

### Environment variables
```
# Required for live scan (optional — site works without it)
ANTHROPIC_API_KEY=sk-ant-...

# Optional: invite code to gate live scan access
SCAN_CODE=your-secret-code

# Optional: set to 'true' to disable live scan entirely
STATIC_ONLY=false
```

## Aesthetic direction: CRT PHOSPHOR TERMINAL

- **Primary**: Green phosphor on black (#33ff33 on #0a0a0a)
- **Scanlines**: CSS repeating-linear-gradient overlay, subtle but present
- **Fonts**: VT323 for headers/display, IBM Plex Mono for body/data
- **Glow**: Subtle text-shadow green glow on key elements (0 0 10px rgba(51,255,51,0.5))
- **Animations**: Cursor blink, boot sequence on load, flicker effect on new data
- **Category colors**: Amber (transit), Red (food), Pink (sunset/art), Cyan (maps/geo), Green (utility), Purple (AI/ML)
- **NO**: Rounded corners beyond 2-3px, gradients, glassmorphism, shadows, modern UI patterns
- **YES**: Hard borders, monospace everything, terminal prompts, ASCII decorations, blocky layouts
- **Icons**: No icon library — use ASCII/Unicode only ( ◆ ▶ █ ░ ▓ ▒ ■ □ ● ○ ► ◄ ▲ ▼ ╔ ╗ ╚ ╝ ║ ═ ┌ ┐ └ ┘ │ ─ )

## Tech stack
- **Framework**: Next.js 14 (App Router, TypeScript)
- **Styling**: Tailwind CSS v3 with custom CRT theme
- **AI Engine**: Anthropic API (`claude-sonnet-4-20250514`) + web_search tool
- **Rate Limiting**: In-memory rate limiter (no external deps)
- **Cache**: In-memory Map with TTL for live scan results
- **Static Data**: JSON files in `/data/` directory, generated by scan script
- **Fonts**: Google Fonts (VT323, IBM Plex Mono)
- **Deployment**: Vercel (free tier)

### 1. Types (`lib/types.ts`)
```typescript
export interface Project {
  id: string;
  title: string;
  author: string;
  description: string;
  category: Category;
  url: string;
  source: Source;
  date: string;
  likes?: number;
  city: CityKey;
  deepScan?: DeepScanData;
}

export type Category = 'TRANSIT' | 'FOOD' | 'SUNSET' | 'MAPS' | 'UTILITY' | 'AI' | 'ART' | 'OTHER';
export type Source = 'twitter' | 'github' | 'reddit' | 'hackernews' | 'blog' | 'producthunt' | 'other';
export type CityKey = 'nyc' | 'london' | 'tokyo' | 'berlin' | 'paris' | 'sf' | 'seoul' | 'mumbai' | 'lagos' | 'sao_paulo';

export interface DeepScanData {
  githubStars?: number;
  techStack?: string[];
  lastUpdated?: string;
  summary?: string;
  vibeScore?: number;
}

export interface CityConfig {
  key: CityKey;
  name: string;
  displayName: string;
  gridName: string;
  searchTerms: string[];
  timezone: string;
}

export interface DiscoverRequest {
  city: CityKey;
  category?: Category;
  query?: string;
  scanCode?: string;
}

export interface DiscoverResponse {
  projects: Project[];
  meta: {
    city: string;
    total: number;
    cached: boolean;
    source: 'static' | 'live';
    timestamp: string;
  };
}
```

### 7. Live scan API route (`app/api/discover/route.ts`)

```typescript
// POST /api/discover
// Body: DiscoverRequest
//
// 1. If STATIC_ONLY=true, return 403 "LIVE SCAN DISABLED"
// 2. If SCAN_CODE is set, validate request.scanCode
// 3. Rate limit check (5/min per IP)
// 4. Check cache — if hit, return cached
// 5. Call Anthropic API with web_search tool:
//    model: 'claude-sonnet-4-20250514'
//    tools: [{ type: 'web_search_20250305', name: 'web_search' }]
// 6. Parse JSON, deduplicate, generate IDs
// 7. Cache results (30 min)
// 8. Return DiscoverResponse with source: 'live'
//
// Terminal-themed errors:
// 403: "ACCESS DENIED — INVALID SCAN CODE"
// 429: "RATE LIMIT — COOL DOWN 60s"
// 500: "SIGNAL LOST — SCANNER MALFUNCTION"
```
